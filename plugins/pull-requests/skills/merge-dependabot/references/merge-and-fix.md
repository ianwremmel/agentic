# Merge & Fix: Multi-Branch Merge Algorithm

This reference details the algorithm for merging multiple Dependabot PR branches
into the combined branch and resolving any issues.

## Prerequisites

- `WORKTREE_DIR` is set and contains a clean worktree based on `origin/main`
- `CANDIDATE_PRS` is a list of passing Dependabot PRs, each with `number`,
  `title`, and `headRefName`
- Working directory is `$WORKTREE_DIR`

## Step 1: Fetch All Candidate Branches

Fetch all candidate branches in a single call:

```bash
cd $WORKTREE_DIR && git fetch origin branch1 branch2 ... branchN
```

Replace `branch1 ... branchN` with the `headRefName` values from
`CANDIDATE_PRS`.

## Step 2: Merge Each PR

Process `CANDIDATE_PRS` in order by PR number (oldest first):

For each PR:

1. Attempt the merge:

    ```bash
    cd $WORKTREE_DIR && git merge --no-edit origin/<headRefName>
    ```

2. **Clean merge** (exit code 0): Add to `MERGED_PRS`, continue to next PR.

3. **Merge conflict** (exit code non-zero): Check which files conflict:

    ```bash
    cd $WORKTREE_DIR && git diff --name-only --diff-filter=U
    ```

4. **Evaluate conflicts**:

    a. If only `package-lock.json` and/or `package.json` files conflict:
    - **`package-lock.json`**: Accept ours — the lockfile will be regenerated by
      `npm install` after all merges:

        ```bash
        cd $WORKTREE_DIR && git checkout --ours package-lock.json && git add package-lock.json
        ```

    - **`package.json`**: These are typically adjacent-line conflicts from
      parallel dependency version updates. Dispatch a `senior-engineer` subagent
      to resolve each conflicting `package.json` by accepting the version bumps
      from **both** branches (keep all updates). The engineer should:
        1. Read the conflicting file(s)
        2. Resolve conflict markers by keeping the higher version from each
           dependency update
        3. `git add` the resolved file(s)

    - After resolving all conflicts, complete the merge:

        ```bash
        cd $WORKTREE_DIR && git commit --no-edit
        ```

    - Add to `MERGED_PRS`, continue.

    b. If **any source files** (non-`package.json`, non-`package-lock.json`)
    conflict:
    - Abort the merge:
        ```bash
        cd $WORKTREE_DIR && git merge --abort
        ```
    - Add to `SKIPPED_PRS` with reason "source file conflicts"
    - Continue to next PR.

## Step 3: Regenerate Lockfile and Build

After all merges are complete:

1. Regenerate the lockfile:

    ```bash
    cd $WORKTREE_DIR && npm install
    ```

2. Build the project:

    ```bash
    cd $WORKTREE_DIR && npm run build
    ```

3. If the build **succeeds**, continue to Step 4.

4. If the build **fails**, dispatch a `senior-engineer` subagent to fix the
   incompatibilities. Provide the engineer with:
    - The full build error output
    - The list of merged dependency updates (from `MERGED_PRS`) for context
    - The worktree path (`$WORKTREE_DIR`)
    - Instructions to fix type errors, breaking API changes, or other
      incompatibilities introduced by the combined dependency updates

    After the engineer completes, re-run `npm run build`. Maximum 3 fix
    attempts. If still failing after 3 attempts, escalate to the user.

## Step 4: Commit Generated Changes

After a successful build, commit any generated file changes (lockfile updates,
codegen output, etc.):

```bash
cd $WORKTREE_DIR && git add -A && git diff --cached --quiet || \
  git commit -m "chore(deps): update generated files after dependency merge"
```

## Summary Report

After completing the merge phase, report:

```
Merge results:
- Merged: N PRs (#1 title, #2 title, ...)
- Skipped: M PRs (#3 title — reason, ...)
```
